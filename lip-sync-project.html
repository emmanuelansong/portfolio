TheLip SyncCharacter

A Unity-based lip-sync character project featuring adjustable blendshapes and sliders to control the character’s emotions. Utilizes Microsoft Azure to analyze audio and generate visemes, allowing the character’s mouth to match speech in real-time. Demonstrates real-time audio-to-animation integration and expressive character control.

What this project is about?

Summary

This project focuses on building a believable speech and facial animation system for a 3D game character. The objective was to design a real-time lip-synchronisation pipeline capable of translating audio or synthesised speech into accurate facial movement using phoneme-to-viseme mapping. Developed in the Unity game engine, the solution combines speech synthesis, blendshape-driven animation, and pre-authored body gestures to enhance character realism and player immersion. The system demonstrates how lightweight technical workflows can be used to create expressive dialogue interactions without relying on complex motion-capture pipelines, making it suitable for indie games and interactive media experiences.

My Role

I worked as the sole developer, responsible for the full project lifecycle including research, system design, implementation, and evaluation. My work involved developing the speech-driven animation system using C# scripting, integrating Microsoft Azure Text-to-Speech services, configuring facial blendshape rigs, and implementing animation sequencing within Unity. I also handled UI prototyping, 3D asset integration, playtesting, performance optimisation, and iterative feature improvements based on feedback. Additionally, I managed project planning using structured development methodologies and ensured the system remained modular and adaptable to different character models.

Core Tools

Key Outcome

The final result was a fully functional real-time lip-synchronisation prototype capable of driving facial animation directly from speech input inside a live game environment. The system achieved accurate phoneme-to-viseme mapping, improved character believability, and demonstrated a stable animation pipeline that integrates smoothly with audio playback and character motion. Through iterative testing and refinement, the project delivered a modular dialogue animation framework that can be reused across multiple characters and scenarios, highlighting the potential of combining speech synthesis technology with real-time game animation systems to create more immersive interactive experiences.

Systems & highlights

Gameplay Core Loop
Speech-driven character animation

The central interaction system allows a 3D character to deliver dialogue with synchronised facial animation in real time. When a line of dialogue is triggered, the system converts text or audio input into speech output while dynamically driving mouth movement using viseme data. This creates a more believable conversational experience compared to traditional pre-baked animations, improving immersion and allowing dialogue sequences to feel responsive and expressive. Supporting body gestures and subtle facial motion further reinforces the sense of a living character within the game world.

Tech Architecture
Real-time lip-sync animation pipeline

The system is built around a modular animation pipeline developed in C# within Unity. Dialogue text is processed through Microsoft Azure’s Text-to-Speech service, which generates both synthesised audio and timestamped viseme data. This data is then interpreted by custom scripts that drive blendshape-based facial rigs and synchronise animation playback using the Unity Timeline system. Object-oriented programming principles were used to structure reusable animation controllers, audio managers, and UI interaction logic. The architecture was designed to remain adaptable to different character models and easily extended with additional facial expressions or procedural animation features.

UX Polish
Expressive feedback & usability design

To improve player readability and overall presentation, the project incorporates clear dialogue triggering controls, simple UI interaction, and visual feedback from facial motion. Subtle animation details such as randomised brow movement, eye focus behaviour, and gesture-based body animation help prevent the character from appearing static or robotic. Iterative playtesting and supervisor feedback informed adjustments to animation timing, speech pacing, and interface layout. These refinements improved the natural flow of dialogue scenes and contributed to a more engaging interactive experience.

How I built it

Focus Areas

Real-time character animation, speech synthesis integration, blendshape facial rigging, modular system architecture, and dialogue interaction usability.

Phase 1
Planning & research
The project began with investigating existing lip-synchronisation techniques used in games and animation pipelines. I analysed approaches based on phoneme recognition, procedural animation, and motion-capture workflows to determine a realistic scope for a solo technical project. Early design goals focused on creating a lightweight real-time solution that could function within a game engine using open-source tools. Unity was selected as the primary development environment due to performance, documentation support, and familiarity with C# scripting workflows.
Phase 2
Prototype & iteration
Initial prototypes focused on building a speech-driven animation pipeline capable of converting dialogue into viseme-based facial motion. Microsoft Azure’s Text-to-Speech API was integrated to generate synthesised audio alongside timestamped viseme data, which was then mapped to blendshape animations within Unity. Iterative testing helped refine animation timing, dialogue pacing, and overall system responsiveness. Additional improvements included integrating pre-authored body gestures and subtle facial movements to enhance the character’s visual believability.
Phase 3
Challenges & solutions
One of the main technical challenges involved achieving reliable lip-sync accuracy between audio playback and facial animation. Early builds experienced timing delays and unnatural mouth transitions. This was addressed by implementing Speech Synthesis Markup Language (SSML) adjustments and restructuring the animation update logic to ensure more precise synchronisation. Additional challenges included hardware performance limitations and compatibility issues when experimenting with alternative systems such as facial tracking integrations. These constraints influenced design decisions towards a more optimised and modular architecture that could run consistently on lower-spec development machines.
Phase 4
Outcome & next steps
The final outcome was a working real-time dialogue animation prototype capable of producing believable facial motion driven directly by synthesised speech. The project demonstrated how combining speech technologies with game animation systems can improve character expressiveness while maintaining a practical production scope. Future improvements would focus on implementing emotion-driven facial states, expanding the system with procedural animation blending, and testing higher-fidelity character models to further enhance realism and presentation quality.
