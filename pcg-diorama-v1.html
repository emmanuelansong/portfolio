ProcedurallyGeneratedDiorama

A procedurally generated environment study built in Unity that explores how terrain generation, splatmapping, vegetation placement, and structure layout systems can be combined to create a small but visually directed diorama. The project is interesting as a PCG case study because it balances algorithmic generation with environment art decision-making, using technical systems to support a specific world theme rather than producing a purely random landscape.

What this project is about

Summary

The Procedurally Generated Diorama V1.0 is a procedural environment prototype centred on building a themed scene through layered generation systems. The project began with historical and artistic research, initially exploring a mix of ancient and modern visual influences, before narrowing into a stronger concept built around a Japanese and Mayan inspired landscape. The final scene combines terrain formation, terrain texturing, tree and rock generation, and building placement experiments to create a compact environment that feels curated instead of random. It was designed as both a technical PCG study and an environment storytelling exercise.

My Role

I developed the project as a solo environment and procedural generation study, handling the full workflow from reference gathering and theme development through to system implementation, visual iteration, and scene refinement. My work included building the terrain with LibNoise, researching and implementing improved splatmapping techniques, evaluating multiple approaches for vegetation distribution, and exploring graph-based procedural layout methods for building placement using concepts such as Delaunay triangulation and Prim's Algorithm.

Core Tools
Unity C# LibNoise Terrain Splatmapping Vegetation Generator Delaunay Triangulation Prim's Algorithm
Key Outcome

The project resulted in a cohesive procedural diorama that demonstrates how multiple generation systems can work together to support visual composition, environment density, and world theme. Rather than treating procedural generation as a standalone feature, the final outcome shows how it can be used to create a scene with stronger readability, atmosphere, and environmental identity.

Systems & highlights

Worldbuilding Concept
Theme-driven procedural environment design

The project was shaped around the idea that a procedural scene should still feel authored. Early concept work explored mixing ancient and modern architecture, but this created visual and structural problems when trying to place flatter buildings into a mountainous environment. The direction was then refined into a more cohesive Japanese and Mayan inspired landscape. This shift made the generation work more focused. Every system, from landform shape to vegetation density, was used to reinforce a clear environmental identity rather than generating content for its own sake.

Tech Pipeline
Layered generation workflow

The diorama was built through a layered pipeline of terrain generation, material blending, vegetation placement, and layout experimentation. Terrain form was created using LibNoise, blending different functions together until the landmass felt believable and appropriate for the theme. Surface detail was then improved using a more refined splatmapping approach after replacing an earlier, less optimal texturing method. Trees and rocks were distributed using a Vegetation Generator workflow, giving better artistic control over variables like height, slope, and radius.

Iteration Experimentation
Comparing algorithms for better visual results

A major part of development was comparing techniques not just by implementation difficulty, but by how they looked in the final scene. Poisson Disk Sampling was initially considered for tree placement, but it created spacing that felt too open for the atmosphere I wanted. Replacing it with a more controllable vegetation workflow produced denser and more convincing results. Building placement followed a similar process. I experimented with a dungeon-style procedural layout that generated cuboids, separated overlaps, and connected them using Delaunay triangulation and minimum spanning tree logic. Even where these systems did not become the final visual direction, they were valuable in shaping the project's broader PCG exploration.

// development process

How I built it

Focus Areas

Terrain generation, splatmapping, vegetation placement, structural layout logic, reference-driven worldbuilding, and visual iteration.

Phase 1
Initial research & direction
The project started with reference gathering and theme exploration. I wanted the diorama to connect to ancient history, and initially explored a blend of ancient and modern societies. References were gathered from Pinterest, games like For Honor and Ghost of Tsushima, and real-world historical imagery. During this phase I identified a major design issue: many architectural ideas suited flatter landscapes, while the world I was building naturally leaned toward a more mountainous terrain structure. This led to a clearer visual direction and a stronger combined theme built around Japanese and Mayan influences.
Phase 2
Generating terrain & materials
Terrain generation was built using LibNoise, an open-source library that provided the core functions for shaping the landscape. This phase involved a lot of trial and error, blending different functions together until the terrain pattern felt both believable and visually appropriate for the diorama. Once the landform was working, I moved on to terrain splatmapping. My first approach used a procedural splatting workflow, but I later replaced it with a more refined method inspired by further research, producing a smoother and more convincing material blend.
Phase 3
Trees, rocks & building placement
The next phase focused on making the scene feel populated and believable. I first explored Poisson Disk Sampling for tree placement, which worked well computationally but did not deliver the density or spacing I wanted visually. I eventually switched to a Unity Vegetation Generator, which gave far more control over parameters such as height, slope, and radius. For structures, I experimented with a procedural dungeon-style building system. This involved generating room-like cuboids, using a separation algorithm to avoid overlap, then connecting them through Delaunay triangulation and Prim's Algorithm to form a linked layout. Even as an experiment, this phase was useful in expanding the project's structural PCG scope.
Phase 4
Outcome & next steps
The final outcome is a procedurally assembled diorama that combines multiple systems into a single themed environment. It demonstrates how procedural techniques can support more than automation, they can also contribute to scene composition, environment mood, and readable worldbuilding. In a future version, I would push the project further through more advanced biome rules, stronger integration between terrain analysis and building placement, and more direct control over the relationship between generated output and a target visual style.
